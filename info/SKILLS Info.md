系统提示词: 全局加载的标准化指令和限制 纯文本实现

SKILL: 作为操作指南, 指定主要的指令  约束 规范, 通过已有的参考做对比 以及最后通过脚本的执行和实现

MCP: 连接外部的资源, 获取所需内容进行处理

# Info

**SKILL三层结构 :**

1. SKILL.MD   核心指令  掌管决策,先做什么后做什么定义行为边界  字数约束大概在 500-2000字
2. REFERENCES 主要参考 掌管标准,什么算好的什么算差的, 按需查阅不占用上下文
3. SCRIPTS 执行脚本 掌管执行, 执行脚本, AI直接执行即可

**编写SKILL的标准:**

1. 核心SKII.md字数约束在 500 - 2000字间
   1. 核心功能描述,推荐一行说清楚
   2. 输入/输出 JSON Schema 一般不超过30行
   3. 异常处理分支
2. 只做专项工作
3. 定制性和交互性
   1. SKILL内部不能存在硬编码,全部通过入参或者LLM 用户动态传入来实现
   2. 交互性: SKILL可以根据条件自动触发或者手动触发,针对些存在风险的执行 需人工确认比如: 删除数据库、转账处理等敏感操作
4. 边界条件和注意事项
   1. 什么条件下不可触发此SKILL
   2. 报错返回什么结构化信息让LLM来进行处理
5. 可组合性, 可以串联其他SKILL
   1. 出参和入参的管道话管理, 例如: SKILLA的出参可以正好对应SKILLB的入参格式,可以根据条件来触发串联
   2. 组合的权利交给LLM处理,或者前端视觉编排器, SKILL自身保证原子性和无状态(只负责处理执行操作,不记录任何以往数据 避免数据污染)
6. 确定性和幂等性
   1. 相同的参数下, SKILL产生的结构是可预测和确定的结果
   2. 在多次执行 "写操作"的情况下,多次执行的结果和一次执行结果副作用相同的;防止大模型多次执行卡顿
7. 安全与权限边界
   1. 每个SKILL都需要最低权限限制(只读 读写 系统管理等)保证其操作不会越界
