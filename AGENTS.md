# ⚖️ SKILL 核心设计与审计标准

1. 【单一职责 (Single Focus)】: 只做专项工作。判断标准：能否用“动词+名词”在 5 个字内描述清楚（如：读取日志、发送邮件）。若包含“以及/并且/然后”，必须拆分。
2. 【高信息密度文档】: 核心 SKILL 说明文档拒绝长篇大论，必须精简在 2 屏以内。强格式化，必须包含：功能描述、标准 JSON Schema、2个典型入参示例。
3. 【高度参数化 (Customization)】: 严禁硬编码。所有可变因子必须暴露为入参，并利用 JSON Schema 限制类型、枚举值及边界。
4. 【自愈性边界 (Robustness)】: 必须显式定义边界条件。任何内部异常必须被捕获，并转化为标准的、能引导大模型自动修正参数的错误文本。
5. 【管道化组合 (Composability)】: 技能本身保持无状态，不直接依赖其他技能；但其输出数据结构必须标准化，以便 LLM 或编排器将其结果作为下一步其他技能的输入。
6. 【执行安全性 (Security)】: 必须声明触发此技能的最低权限，且执行高危、不可逆操作（如删除、支付、破坏性扫描）时，必须具备“人工二次确认（Human-in-the-loop）”的拦截设计。
